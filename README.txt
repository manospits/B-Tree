Περιγραφή Υλοποίησης:
--> Τα μπλοκ ευρετηρίου οργανώνονται ως εξής:
1ο byte ->	block type:	τύπος μπλοκ ('b' για μπλοκ ευρετηρίου) - char
2o byte-2o byte+sizeof(int) ->	counter:	αριθμός κλειδιών που υπάρχουν στο μπλοκ - int
Διάταξη υπόλοιπων bytes του μπλοκ:
	δείκτης|κλειδί|δείκτης|κλειδί|...|δείκτης|κλειδί|δείκτης
, όπου δείκτης είναι ένας ακέραιος που αντιστοιχεί στον αριθμό του μπλοκ
ευρετηρίου/δεδομένων με κατάλληλες τιμές, και κλειδί είναι πεδίο κατάλληλου τύπου 
και μεγέθους ανάλογα με τον τύπο του ευρετηρίου που έχει επιλεγεί κατά τη 
δημιουργία του.

--> Τα μπλοκ δεδομένων οργανώνονται ως εξής:
1ο byte ->	block type:		τύπος μπλοκ ('d' για μπλοκ δεδομένων) - char
Ακέραιος -> 	next block id:		ακέραιος που αντιστοιχεί στο επόμενο (δεξιά) 
κατά σειρά μπλοκ δεδομένων - int
Ακέραιος -> 	record counter:		αριθμός πλειάδων που υπάρχουν στο μπλοκ - int
Διάταξη υπόλοιπων bytes του μπλοκ:
	Πεδίο1Πεδίο2|Πεδίο1Πεδίο2|...|Πεδίο1Πεδίο2
, όπου το είδος και το μέγεθος των πεδίων είναι αντίστοιχα με τις επιλογές κατά τη 
δημιουργία του ευρετηρίου.

-->Το header block κάθε αρχείου ευρετηρίου περιλαμβάνει τα εξής (με την ίδια σειρά):
-Τη συμβολοσειρά "AM_FILE_HEADER"
-To μέγεθος ένος ακεραίου στο σύστημα
-Τον τύπο του πρώτου πεδίου
-Το μήκος του πρώτου πεδίου
-Τον τύπο του δεύτερου πεδίου
-Το μήκος του δεύτερου πεδίου
-Τον αριθμό (αναγνωριστικό) του μπλοκ-ρίζα
-Τον αριθμό (αναγνωριστικό) του αριστερότερου μπλοκ δεδομένων

Σχόλια Υλοποίησης:
--Στο πρώτο μπλοκ του αρχείου, αμέσως μετά την αναγνωριστική συμβολοσειρά των αρχείων
AM, φυλάσσεται το μέγεθος του ακεραίου του συστήματος στο οποίο δημιουργήθηκε το ευρετήριο.
Κατά το άνοιγμα του ευρετηρίου, αν το σύστημα στο οποίο ανοίγεται έχει διαφορετικό μέγεθος
ακεραίου, τότε δεν ανοίγει το ευρετήριο και επιστρέφεται μήνυμα λάθους, καθώς αν 
συνεχιστεί η επεξεργασία θα υπάρξουν σφάλματα, αφού στις συναρτήσεις για τον υπολογισμό
των θέσεων μνήμης χρησιμοποιείται η συνάρτηση sizeof().

--Μία θέση στους πίνακες AM_files και index_scans θεωρούμε ότι είναι κενή αν το πεδίο
id και am_file_pos αντίστοιχα είναι -1. (Οι πίνακες αυτοί είναι πίνακες από structs).

--Μέχρι ο αριθμός των πλειάδων στο αρχείο να ξεπεράσει το μέγιστο αριθμό πλειάδων που
χωράνε σε ένα μπλοκ, στο αρχείο υπάρχει μόνο ένα μπλοκ δεδομένων και κανένα μπλοκ 
ευρετηρίου (και θεωρείται η "ρίζα" του δέντρου). Όσο αυξάνονται οι εισαγωγές, αρχίζουν
να δημιουργούνται τα μπλοκ ευρετηρίου.

--Αν δεν έχει γίνει καμία εισαγωγή στο αρχείο τότε το id της ρίζας στο block πληροφορίας
είναι ίσο με -1 και παίρνει, οταν κλήθει για πρώτη φορά η εισαγωγή, το id του
δημιουργηθέντα block.

--Πριν δημιουργήθει κάποιο index block υπάρχει μόνο ενα data block με δείκτη σε επόμενο
block ίσο με -1, το οποίο η διασπασή του όταν αυτό γεμίσει θα οδηγήσει στην δημιουργία
ενός index block.

--Σε περιπτώση που η διάσπαση του block χωρίζει σε δυο block όμοια κλειδία η διάσπαση
γίνεται με τον κατάληλλο τρόπο ώστε τα όμοια κλειδιά να βρέθουν τελίκα στο ίδιο block.
